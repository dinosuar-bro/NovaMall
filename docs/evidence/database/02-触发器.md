## 商品改价直接 DML

执行 SQL 摘要：

```sql
SET @novamall_actor_user_id = 4;
SET @novamall_request_id = '00000000-0000-4000-8000-000000000603';

SELECT id, price, stock, status FROM products WHERE id = 3;
UPDATE products SET price = price + 0.01 WHERE id = 3;
UPDATE products SET price = price - 0.01 WHERE id = 3;

SELECT id, product_id, old_price, new_price, changed_by, request_id
FROM product_price_history
WHERE product_id = 3
ORDER BY id DESC
LIMIT 4;

SELECT id, actor_user_id, request_id, table_name, record_id, action,
       JSON_EXTRACT(old_data, '$.price') AS old_price,
       JSON_EXTRACT(new_data, '$.price') AS new_price
FROM audit_logs
WHERE table_name = 'products'
  AND record_id = 3
ORDER BY id DESC
LIMIT 4;
```

实际输出：

```text
id  price  stock  status
3   19.90  19     PUBLISHED

id  product_id  old_price  new_price  changed_by  request_id
2   3           19.91      19.90      4           00000000-0000-4000-8000-000000000603
1   3           19.90      19.91      4           00000000-0000-4000-8000-000000000603

id   actor_user_id  request_id                              table_name  record_id  action  old_price  new_price
132  4              00000000-0000-4000-8000-000000000603    products    3          UPDATE  "19.91"   "19.90"
131  4              00000000-0000-4000-8000-000000000603    products    3          UPDATE  "19.90"   "19.91"
```

结论：

- `trg_products_price_history` 自动记录两次价格变更；
- `trg_products_audit` 自动记录商品更新审计；
- 审计操作者和 requestId 来自连接级变量；
- 改价后价格已改回 19.90，证据采集没有改变商品最终价格。
