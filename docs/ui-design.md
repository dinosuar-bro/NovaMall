# NovaMall 前端与交互设计

## 1. 设计目标

NovaMall 的视觉主题为“邻里星市”：以纯白和清醒中性表面承载内容，暖珊瑚用于主要操作和当前状态，辅以少量清泉青绿。品牌气质亲和、温暖、活力，但不使用奶油米色、促销式高饱和堆叠或与任务无关的复杂动画。根目录 `PRODUCT.md` 和 `DESIGN.md` 是后续前端实现的权威设计依据。

## 2. 三套布局

### 会员工作区

- 桌面端侧边栏：商品目录、购物车、订单列表；
- 商品目录：关键词、分类筛选、商品卡片和加购反馈；
- 购物车：地址表单、结算地址、商品数量自动更新、删除和结算确认；
- 订单列表：模拟支付和确认收货。

### 店主后台

- 桌面端侧边栏：商品管理、订单履约；
- 顶栏显示当前店主账号；
- 商品管理表单使用稳定两列布局，商品简介独占整行，图片和创建按钮底部对齐；
- 商品列表展示共享商品池中的商品，不按店铺隔离；
- 订单履约只展示共享履约子订单和允许发货的操作。

### 系统后台

- 侧边栏：分类管理、开店审核、数据库证据；
- 审核页面展示申请内容、申请人和审批操作；
- 数据库证据页面只读展示审计日志和有效销量 Top 10。

## 3. 页面清单

已登录工作区使用同一个应用壳：左侧为当前角色的功能导航，顶部右侧为统一默认头像。点击头像只展示两个操作：进入个人主页、退出登录。系统不开放头像上传，所有用户统一使用平台默认头像。

### 公开与会员页面

- 注册、登录；
- 商品目录：`/member/catalog`；
- 开店申请：`/member/applications`；
- 购物车：`/member/cart`；
- 订单列表：`/member/orders`；
- 个人主页：`/profile`，可修改展示名、手机号和密码；修改密码需要确认新密码，旧手机号无法解密时手机号框为空并允许重新填写；
- 地址保存集成在购物车页。

### 店主页面

- 商品管理：`/owner/products`；
- 订单履约：`/owner/orders`。

### 管理员页面

- 分类管理：`/admin/categories`；
- 开店申请列表和审核：`/admin/applications`；
- 数据库证据：`/admin/database`。

## 4. 关键交互

### 购物车

按商品列表展示，会员一次结算当前购物车。数量修改使用乐观界面反馈，服务端失败时回滚并显示原因。

### 结算

展示结算地址、商品列表和总金额。提交前弹出“确认结算明细”，列出商品名、数量、单价、小计和总价。确认后生成并复用 `checkoutToken`；请求进行期间禁用重复按钮。网络失败后重试仍使用同一 Token。

### 订单

总订单展示支付和整体状态，子订单展示履约状态。只有当前状态允许的操作才显示，但后端仍执行完整校验。退款只保留数据库专项测试路径，不作为当前前端主线页面。

### 表单

- 字段旁显示可操作的校验提示；
- 服务端字段错误映射回对应输入；
- 明显不合法的输入先在前端拦截，避免用户提交后只看到通用失败；
- 普通用户界面不直接展示技术错误码、堆栈、SQL 或 requestId；需要排查时由日志和响应头关联请求编号；
- 未保存离开时提示；
- 提交期间显示明确状态并避免重复提交。

### 角色导航

侧边栏按当前工作区显示对应导航项。会员工作区只展示会员相关入口，店主工作区只展示共享商品和履约入口，管理员工作区只展示平台管理入口。前端导航只改善体验，后端角色校验仍是最终安全边界。

根路径根据当前会话进入用户拥有的最高优先级工作区：`ADMIN -> /admin/categories`、`OWNER -> /owner/products`、`MEMBER -> /member/catalog`。角色工作区在渲染业务内容前必须先确认会话；未登录用户回到登录页，不在业务页面显示认证错误或请求编号。

工作区统一按桌面端使用居中的主内容容器，阶段面板、表单输入区、主按钮和状态提示应在同一面板宽度内对齐，不再针对移动端做额外优化。

## 5. 状态设计

每个数据页面必须具备：

- 加载状态；
- 空状态；
- 错误状态与重试；
- 权限不足；
- 数据已被并发修改的冲突提示；
- 操作成功反馈。

## 6. 可访问性

- 所有交互可通过键盘完成；
- 使用语义化 HTML 和可见焦点；
- 表单标签与输入正确关联；
- 状态不只依赖颜色；
- 文字和背景满足合理对比度；
- 图片提供替代文本；
- 动画尊重 `prefers-reduced-motion`。

## 7. 桌面端边界

- 会员、店主和管理员界面统一面向常见桌面宽度；
- 窄屏只要求内容不出现严重遮挡，不提供完整移动端交互优化；
- 不为任一角色制作独立移动应用。

## 8. 前端数据边界

- 不在前端保存密码、AES 密钥或数据库信息；
- 不使用 localStorage 保存身份令牌；
- API BIGINT ID 保持字符串；
- 金额以字符串接收并通过金额工具格式化；
- 服务端返回 DTO，不直接绑定数据库字段对象。
