# AGENTS.md

## 交流与授权

- 全程使用中文交流，包括注释、文档、错误提示和提交说明。
- 任何文件修改、创建、删除、依赖安装、配置更改或 Git 写操作前，必须获得用户明确授权。
- 优先复用成熟 skill、MCP 和开源工具；需要安装新工具时先说明用途并取得授权。

## 文档先行

- 编写或修改代码前，必须先明确并更新该阶段的需求、接口、数据结构、实现方案和验收标准。
- 每个开发阶段在 `docs/development-plan.md` 中有独立最小闭环。
- 阶段文档经用户确认后才能开始该阶段编码。
- 实现行为与文档冲突时先修正文档并重新确认，不能静默偏离。

## 技术约束

- 包管理器统一使用 pnpm，不使用 npm 或 yarn。
- 前后端统一使用 TypeScript，禁止新增 JavaScript 业务文件。
- TypeScript 开启严格模式，禁止 `any`；确有必要时必须写明原因和边界。
- 前端使用 React，后端使用 Express，数据库使用 MySQL。
- 数据访问使用 mysql2 和显式 SQL，不引入 ORM。
- 密码只能使用安全的不可逆哈希；手机号按安全文档使用 AES 加密。
- 数据库金额统一使用 `DECIMAL`，禁止用浮点数表示金额。

## 当前项目状态

- 当前已完成到阶段 3 商品目录闭环：认证、个人主页、商户入驻、分类管理、店主商品发布、公开商品列表和图片上传均已有实现。
- 前端功能已拆到独立路由，会员、店主、管理员通过侧边导航切换；个人主页位于 `/profile`。
- Docker 前端入口为 `http://localhost:8080`，主库 MySQL 为本地调试映射到 `127.0.0.1:3307`，测试库映射到 `127.0.0.1:3308`。
- 后端上传目录挂载到 Docker 卷 `uploads-data`，商品图片路径为 `/uploads/products/...`，前端访问时经 `/api/v1/uploads/...` 代理；历史图片缺失时显示 `product-placeholder.svg`。
- 默认头像为 `apps/web/public/default-avatar.svg`，系统不向用户开放上传头像。
- `.env`、上传目录、测试结果、构建产物和 pnpm 本地 store 都不得提交。

## 架构边界

- 后端调用方向固定为 `Route → Controller → Service → Repository`。
- Route 只声明路由和中间件；Controller 只处理 HTTP；Service 执行业务规则；Repository 是应用层唯一允许访问数据库的位置。
- 不允许 Controller 直接执行 SQL，不允许 Repository 决定 HTTP 状态码。
- 会员、店主和管理员权限必须在后端校验，不能依赖前端隐藏按钮。
- 店主共享商品池和履约数据，不再按店铺隔离商品；会员只能访问自己的地址、购物车、订单和申请。

## 数据库规则

- 所有结构变更必须通过版本化 SQL 迁移完成，禁止只在本地手工改表。
- 迁移脚本必须同时考虑主键、外键、唯一约束、检查约束和索引。
- 结算、支付、取消和退款等多表写操作必须使用事务。
- 库存并发必须使用行锁和条件校验，禁止先查库存再无条件扣减。
- 存储过程、触发器、视图、全文索引和审计逻辑必须有独立测试与卸载说明。
- 性能对比必须保存真实数据规模、SQL、执行计划和计时结果，不得虚构。

## 修改原则

- 只修改当前任务必要的文件，不顺手重构或格式化无关代码。
- 优先选择最简单的正确方案，不提前实现后续阶段功能。
- 新增代码产生的无用导入、变量和文件应由本次修改清理。
- 不删除用户已有改动；发现无关问题只报告，不擅自处理。

## 验证规则

- 功能或缺陷必须先定义可验证用例，再实现。
- 每个阶段至少执行类型检查、Lint、单元测试、集成测试和构建；涉及数据库时必须执行数据库专项测试。
- 阶段验收以 `docs/testing-and-acceptance.md` 和 `docs/development-plan.md` 为准。
- 未运行验证时不得声称功能完成或问题已修复。
- Docker 镜像内静态资源不会因 `docker compose restart` 自动更新；涉及前端构建或后端镜像内容时，验证后需要使用 `docker compose up --build -d` 让本地页面生效。
- 集成测试默认依赖 `docker compose -f docker-compose.test.yml up -d mysql-test` 和 `TEST_DATABASE_URL='mysql://novamall:novamall_test_password@127.0.0.1:3308/novamall_test'`。

## Git 规则

- 提交前检查变更范围和 `git diff --check`。
- 不提交密钥、真实手机号、数据库数据卷、上传文件或本地临时产物。
- 未经明确授权，不提交、推送、创建分支或创建 Pull Request。
