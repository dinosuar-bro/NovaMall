# Design System: NovaMall（星选）

## Overview

**Creative North Star: "邻里星市"**

NovaMall 像一座秩序清楚的社区市集：会员可以轻松发现商品，店主知道下一步该处理什么，管理员能准确判断系统状态。参考 Airbnb 的亲和市场体验、Apple Store 的商品层级和克制动效，以及 Shopify Admin 的可信任务流。

产品以清醒的浅色工作环境为主，品牌温度由暖珊瑚色、友好中文文案和及时反馈承担，而不是奶油米色表面。管理界面保持熟悉、紧凑和可靠，商城界面更舒展，但三类界面共享同一组件语言。

**Key Characteristics:**

- 亲和但不幼稚；
- 明快但不喧闹；
- 熟悉的标准控件与明确状态；
- 响应式、可访问、可减少动效；
- 温暖品牌色仅服务重点操作和当前状态。

## Colors

采用 Restrained 策略：中性表面占主导，暖珊瑚品牌色控制在单屏约 10% 以内，辅色只承担链接、信息和状态区分。

### Primary

- **邻里珊瑚**：`oklch(0.55 0.18 35.8)`；用于主要按钮、当前导航和关键焦点。悬停态为 `oklch(0.50 0.19 35.8)`。

### Secondary

- **清泉青绿**：当前阶段只在语义 token 中保留，暂不铺开使用，避免认证界面过度上色。

### Neutral

- **纯白背景**：`oklch(1 0 0)`，页面主背景使用无色度纯白，不使用奶油、米色或羊皮纸色。
- **浅层表面**：`oklch(0.982 0.004 35.8)`，用于侧栏、工具条和表单分区。
- **暖墨正文**：`oklch(0.225 0.026 35.8)`。
- **次级文字**：`oklch(0.405 0.026 35.8)`。

**The Warmth Lives in the Accent Rule.** 温暖只由品牌色、图片和语言承担；禁止同时把背景染成暖奶油色。

## Typography

**Display Font:** `ui-sans-serif, -apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Microsoft YaHei", sans-serif`
**Body Font:** 与 Display 同一家族

**Character:** 字形友好、开放、易读，不使用科技感几何字体或装饰性展示字体。产品界面采用紧凑但不拥挤的固定字号层级。

### Hierarchy

- **Display**：仅用于登录/注册欢迎语和商城关键标题，不超过 3rem。
- **Headline**：页面主标题，稳定、直接、避免夸张字距。
- **Title**：表单分组、空状态和面板标题。
- **Body**：默认阅读文本，正文行长限制在 65–75ch。
- **Label**：表单、按钮和数据标签，不使用全大写宽字距。

**The One Family Rule.** 产品界面只使用一个主要无衬线字体家族，通过字重和字号建立层级。

## Elevation

默认使用色调分层和边界建立结构。阴影只在浮层、菜单和交互提升状态中出现，静态表单与面板不同时叠加宽阴影和细边框。

**The Flat-by-Default Rule.** 静态表面保持平整；用户触发浮层或悬停提升时才出现短距离、低模糊阴影。

## Components

Stage 1 将建立按钮、文本输入、密码输入、状态提示、认证面板、顶部导航和三类角色空壳。所有组件必须具备默认、悬停、焦点、按下、禁用、加载和错误状态。

### Buttons

- 主要按钮使用邻里珊瑚填充和白色文字，形状清楚而不过度圆润。
- 次要按钮使用中性表面与明确边界，幽灵按钮只用于低优先级操作。
- 动效为 150–220ms 的状态反馈，不使用弹跳或弹性缓动。

### Cards / Containers

- 认证面板和空状态只在确有分组语义时使用容器。
- 圆角上限 16px，不创建层层嵌套卡片。
- 同一元素不同时使用 1px 装饰边框和大范围软阴影。

### Inputs / Fields

- 输入框使用清晰标签、稳定高度和可见焦点环。
- 占位符达到正文可读对比度，但不能代替标签。
- 错误状态同时提供图标、文字和 `aria-describedby` 关联。

### Navigation

- 会员使用轻量顶部导航；店主与管理员使用熟悉的侧栏/顶栏结构。
- 当前项由品牌色、字重和形状共同指示，不能只依赖颜色。
- 窄屏保持标准菜单行为，不发明非标准导航控件。

## Do's and Don'ts

### Do:

- **Do** 让主要操作、当前状态和焦点共享一致的暖珊瑚语言。
- **Do** 使用标准表单、按钮和导航行为，减少学习成本。
- **Do** 为加载、错误、成功、空状态和权限不足提供具体下一步。
- **Do** 使用 GSAP 的 transform、autoAlpha 和短促 ease-out 表达状态变化。
- **Do** 通过 `useGSAP`、作用域 ref 和清理机制管理 React 动画。

### Don't:

- **Don't** 使用“双十一式高饱和促销堆叠”。
- **Don't** 使用“冷硬科技后台”的暗色霓虹和终端化装饰。
- **Don't** 使用奶油米色背景、玻璃拟态、渐变文字或同构卡片网格。
- **Don't** 使用大于 16px 的普通卡片圆角。
- **Don't** 使用弹跳、弹性、无意义循环或冗长页面入场动画。
- **Don't** 让内容默认隐藏并依赖动画脚本才能显示。
