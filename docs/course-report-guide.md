# 《数据库课程设计》报告编写指南

## 1. 报告原则

- 以 NovaMall 的真实实现和实验为依据；
- 不复制开源项目描述，不虚构性能数字；
- 每个设计结论对应表结构、SQL、代码或测试证据；
- 图表注明数据规模、环境和生成方式。

## 2. 建议目录

### 第一章 绪论

- 项目背景；
- 设计目标；
- 业务范围；
- 技术路线。

### 第二章 需求分析

- 会员、店主、管理员角色；
- 功能需求；
- 核心业务规则；
- 非功能需求；
- 用例图或业务流程图。

### 第三章 系统总体设计

- 模块化单体架构；
- 前端、后端和数据库边界；
- 功能模块；
- Docker 部署结构；
- 关键数据流。

### 第四章 数据库概念结构设计

- 实体识别；
- 属性与联系；
- ER 图；
- 共享商品池和统一结算建模说明。

### 第五章 数据库逻辑结构设计

- 关系模式；
- 主键、外键和唯一约束；
- 数据字典；
- 订单快照和受控反范式；
- 1NF、2NF、3NF 分析。

### 第六章 数据库完整性与安全

- 实体、参照、域和用户定义完整性；
- 状态机与金额一致性；
- 密码哈希和手机号 AES 加密；
- 审计和敏感信息保护。

### 第七章 高阶数据库技术实现

每项使用统一结构：

1. 业务问题；
2. 选择该技术的原因；
3. SQL 或对象定义；
4. 应用调用方式；
5. 测试数据和执行步骤；
6. 实际结果；
7. 限制与取舍。

依次介绍存储过程、触发器、视图、索引、事务并发、窗口函数、审计、全文检索和数据加密。

### 第八章 系统实现

- 注册与 Session；
- 商户入驻；
- 商品和全文搜索；
- 购物车和统一结算；
- 支付、履约和退款；
- 管理与审计。

### 第九章 测试与性能分析

- 测试环境；
- 功能测试；
- 权限与安全测试；
- 事务回滚与并发测试；
- 索引前后执行计划；
- 全文检索与 LIKE 对比；
- 缺陷与修复。

### 第十章 部署、总结与展望

- Docker 部署；
- 备份恢复策略；
- 项目成果；
- 当前限制；
- 可选扩展方向。

## 3. 必备图表

- 系统上下文图；
- 部署图；
- 完整 ER 图；
- 统一结算时序图；
- 总订单和子订单状态图；
- 关键表数据字典；
- 索引前后执行计划对比表；
- 并发测试结果表；
- 9 项技术验收矩阵。

## 4. 数据库章节检查清单

- ER 图与实际表一致；
- 每条关系说明基数和外键；
- 范式分析包含受控反范式解释；
- 完整性约束区分数据库约束与应用事务规则；
- 存储过程确有输入和输出参数；
- 触发器由直接 DML 验证；
- 视图被实际 API 使用；
- 索引报告包含真实 `EXPLAIN ANALYZE`；
- 并发测试使用两个独立连接；
- Top 10 使用窗口函数；
- 审计说明“谁、何时、对什么做了什么”；
- 全文检索说明中文 ngram 与 LIKE 语义差异；
- 加密展示密文、正确解密和错误密钥结果。

## 5. 证据引用

报告中的命令和结果应引用 `docs/evidence/database/`。性能表格至少包含：

- Git 提交；
- MySQL 版本；
- 表行数；
- 查询参数；
- 执行次数；
- 实际耗时；
- 执行计划关键字段。

如果某项实验未完成，应明确标注“未验证”，不能用预计结果替代。

Stage 6 证据文件与报告章节对应关系：

| 报告主题 | 证据文件 |
|---|---|
| 存储过程 | `docs/evidence/database/01-存储过程.md` |
| 触发器 | `docs/evidence/database/02-触发器.md` |
| 视图 | `docs/evidence/database/03-视图.md` |
| 索引优化 | `docs/evidence/database/04-索引优化.md` |
| 事务与并发 | `docs/evidence/database/05-事务与并发控制.md` |
| 窗口函数 | `docs/evidence/database/06-窗口函数.md` |
| 审计日志 | `docs/evidence/database/07-审计日志.md` |
| 全文检索 | `docs/evidence/database/08-全文检索.md` |

当前 Stage 6 证据已覆盖 9 项高阶数据库技术。每个证据文件内部都保留环境、数据规模和限制说明；报告引用性能和安全结论时不能把本地确定性数据集结论外推为生产性能。

## 6. 答辩演示顺序

1. 用三类账号说明权限；
2. 完成会员申请开店和管理员审核；
3. 展示共享商品和中文搜索；
4. 演示统一结算和订单快照；
5. 并发抢最后库存；
6. 支付、发货和确认收货；
7. 展示 Top 10 和审计；
8. 依次展示 9 项技术的 SQL 与证据；
9. 使用 Docker 说明可复现部署。

答辩时优先解释业务问题、数据库方案和验证结果，不把时间消耗在框架 API 细节上。
