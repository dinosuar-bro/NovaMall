# 存储过程证据：`sp_checkout_cart`

## 目标

验证统一结算存储过程具备输入参数、输出参数、事务、履约子订单创建、库存扣减、购物车清理和 checkoutToken 幂等能力。

## 环境

- 采集时间：2026-06-28
- Git 提交：`6ca5e42`
- MySQL 版本：`8.4.9`
- 证据用户：`stage6_evidence_*`，本地开发库合成数据，不含真实个人信息。

## 成功与幂等用例

执行 SQL 摘要：

```sql
INSERT INTO users (...) VALUES (...);
INSERT INTO user_roles (user_id, role_id, granted_by) VALUES (@stage6_user_id, 1, NULL);
INSERT INTO addresses (...) VALUES (...);
INSERT INTO cart_items (user_id, product_id, quantity)
VALUES (@stage6_user_id, 3, 1), (@stage6_user_id, 7, 1);

CALL sp_checkout_cart(
  @stage6_user_id,
  @stage6_address_id,
  '00000000-0000-4000-8000-000000000601',
  @order_no
);
SELECT @order_no;

CALL sp_checkout_cart(
  @stage6_user_id,
  @stage6_address_id,
  '00000000-0000-4000-8000-000000000601',
  @order_no_repeat
);
SELECT @order_no_repeat;
```

实际输出：

```text
evidence_user_id  evidence_address_id  checkout_token
155               11                   00000000-0000-4000-8000-000000000601

first_order_no
MOafa3c908730611f193440afda4b47e66

repeat_order_no
MOafa3c908730611f193440afda4b47e66

order_no                              status           total_amount  shop_order_count  item_count  payment_status
MOafa3c908730611f193440afda4b47e66    PENDING_PAYMENT  219.89        2                 2           PENDING

id  stock
3   19
7   9999
```

结论：

- OUT 参数返回总订单号；
- 一个购物车包含多个商品后生成 1 个总订单、2 个履约子订单、2 条订单明细；
- 支付记录自动创建为 `PENDING`；
- 商品 3 和商品 7 库存各扣减 1；
- 使用同一 checkoutToken 重复调用返回同一订单号，没有重复扣库存。

## 库存不足回滚用例

执行 SQL 摘要：

```sql
DELETE FROM cart_items WHERE user_id = @stage6_user_id;
INSERT INTO cart_items (user_id, product_id, quantity)
VALUES (@stage6_user_id, 3, 999999);

SELECT COUNT(*) AS orders_before
FROM master_orders
WHERE buyer_user_id = @stage6_user_id
  AND checkout_token = '00000000-0000-4000-8000-000000000602';

SELECT stock AS stock_before FROM products WHERE id = 3;

CALL sp_checkout_cart(
  @stage6_user_id,
  @stage6_address_id,
  '00000000-0000-4000-8000-000000000602',
  @bad_order_no
);
```

实际输出：

```text
orders_before
0

stock_before
19

ERROR 1644 (45000): OUT_OF_STOCK
```

失败后复查：

```text
orders_after
0

product_id  cart_quantity_after
3           999999

stock_after
19
```

结论：

- 库存不足时存储过程返回稳定业务错误 `OUT_OF_STOCK`；
- 未创建总订单；
- 库存保持 19，不出现负库存；
- 购物车失败项仍保留，证明事务没有半提交。
