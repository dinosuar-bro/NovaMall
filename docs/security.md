# NovaMall 安全设计

## 1. 安全目标

- 防止未授权访问和会员资源越权；
- 防止 SQL 注入、CSRF、会话固定和敏感信息泄露；
- 密码不可逆保存，手机号在数据库中不可直接读取；
- 错误和审计信息可追踪但不泄露内部实现。

## 2. 身份认证

- 密码使用 Node.js 内置 `crypto.scrypt` 异步计算；
- 每个密码使用独立随机盐，编码串包含算法版本、参数、盐和摘要；
- 比较使用恒定时间函数；
- 不记录密码、摘要、盐或登录请求正文；
- 登录成功后轮换 Session ID，防止会话固定；
- 登录失败使用统一提示，不暴露用户名是否存在。

## 3. Session 与 Cookie

- Session 存入 MySQL，不使用默认内存存储；
- Cookie 设置 HttpOnly 和 SameSite=Lax；
- 生产环境设置 Secure；
- Cookie 限定合理 Path，不设置宽泛 Domain；
- 登录、权限变更和敏感操作后更新会话；
- 退出时服务端销毁 Session 并清除 Cookie；
- 每个请求重新检查账号状态和当前角色，不能只信任登录时快照。

## 4. CSRF 与跨域

- 所有状态变更请求必须携带 CSRF Token；
- Token 与 Session 绑定，通过自定义请求头提交；
- 生产环境前后端同源，由 Nginx 反向代理；
- 开发环境 CORS 只允许明确的本地前端 Origin，并允许凭证；
- 不使用 `Access-Control-Allow-Origin: *` 配合 Cookie。

## 5. 授权模型

授权分两层：

1. 角色校验：MEMBER、OWNER、ADMIN；
2. 资源归属校验：会员 ID、订单买家和角色权限。

店主共享维护平台商品，更新商品时 SQL 条件必须包含商品 ID 和版本条件；会员读取订单时必须包含当前用户 ID。前端路由守卫只改善体验，不构成安全边界。

## 6. SQL 安全

- 所有用户输入通过 mysql2 参数占位符绑定；
- 排序字段、表名等不能参数化的标识符使用服务器白名单映射；
- 禁止把用户输入拼接到 SQL；
- 生产应用数据库账号不授予 DROP、ALTER、CREATE USER 或 FILE 权限；
- 数据库迁移使用独立账号；
- 管理员数据库演示页不能接受任意 SQL。

## 7. 手机号加密

- `phone_cipher` 保存 AES 二进制密文，`phone_iv` 保存独立随机 IV；
- 使用 `aes-256-cbc`，禁止 MySQL 默认 ECB 模式；
- 密钥只通过 Secret/环境变量提供，不进入 Git、日志或数据库；
- 只有资料展示和履约场景允许解密；
- 审计与错误日志不记录明文手机号；
- 数据库链路在生产环境使用 TLS。

CBC 不提供原生认证能力。课程实现用于展示数据库对称加密；生产扩展应评估 AES-GCM 和专业密钥管理服务。

## 8. 输入和输出安全

- 使用 Zod 校验请求正文、查询参数和路径参数；
- 字符串限制长度并规范化首尾空白；
- React 默认转义文本，不直接渲染商品简介 HTML；
- 不使用 `dangerouslySetInnerHTML` 显示用户输入；
- 响应 DTO 白名单选择字段，不直接返回数据库整行。

## 9. 图片上传

- 只允许配置的图片 MIME 类型和扩展名；
- 同时检查文件魔数，不能只信任请求头；
- 限制单文件大小和图片尺寸；
- 服务端生成随机文件名，忽略用户原文件路径；
- 上传目录禁止执行脚本；
- 数据库写入失败时清理本次新文件；
- 历史订单图片快照对应的文件不得被商品编辑误删。

## 10. 错误和日志

- 每个请求生成 requestId；
- 客户端只看到稳定错误码和安全消息；
- 生产环境不返回堆栈、SQL、表结构、密钥或环境变量；
- 日志对 Cookie、密码、手机号、密钥和请求正文敏感字段脱敏；
- 审计日志与应用运行日志分离。

## 11. 密钥与配置

禁止提交：

- Session Secret；
- AES Key；
- 数据库密码；
- 真实手机号和生产数据；
- 本地 `.env`；
- TLS 私钥。

仓库只提供 `.env.example`，其中只有变量名和安全说明。

## 12. 安全测试清单

- 未登录访问受保护接口返回 401；
- MEMBER 调用 OWNER/ADMIN 接口返回 403；
- 店主只能在店主角色允许的共享商品和履约范围内操作；
- 会员不能读取其他会员地址和订单；
- SQL 注入载荷被当作普通参数；
- 缺少或错误 CSRF Token 的写请求被拒绝；
- 登录后 Session ID 发生变化；
- 禁用账号现有 Session 的后续请求失败；
- 上传伪装图片、超限文件和路径文件名被拒绝；
- 日志与 API 响应不出现敏感信息。
