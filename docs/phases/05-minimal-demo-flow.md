# Stage 5：最小业务演示闭环规格

## 1. 阶段目标

在 Stage 4 数据库核心和订单 API 的基础上，交付可通过浏览器完成的最小商城演示主线。该阶段聚焦课程展示需要的业务闭环：会员浏览商品、加入购物车、维护收货地址、跨店结算、模拟支付；店主查看本店子订单并发货；会员确认收货；管理员只读查看审计日志和 Top 10 数据库证据。

本阶段不扩展完整电商订单中心，不实现真实支付、物流轨迹、优惠、评价、复杂售后或前端 SQL 实验台。

## 2. 成功标准

- 会员页提供商品加购、地址创建、购物车列表、结算、订单列表、模拟支付和确认收货入口；
- 店主页提供本店子订单列表和待发货子订单发货入口；
- 管理员页提供审计日志和有效销量 Top 10 的只读展示；
- 后端补齐店主发货和会员确认收货 API，并继续按 `Route -> Controller -> Service -> Repository` 分层；
- 发货必须校验当前店主所属店铺，确认收货必须校验当前会员拥有对应总订单；
- 支付、发货和确认收货均使用状态条件或事务，重复或非法状态返回稳定错误；
- 前端不暴露任意 SQL 输入能力，只展示预定义数据库证据摘要；
- 演示数据种子可支撑管理员、会员、两个店主、分类、商品和订单主线；
- API 集成测试、前端组件测试和关键 E2E 主线覆盖阶段 5 最小闭环。

## 3. 非目标

- 订单详情页、订单筛选、分页调优和完整状态时间线；
- 物流公司、物流单号、物流轨迹和收货地址智能识别；
- 退款页面、退款原因、凭证上传、客服介入和退款入账；
- 真实支付网关、支付回调、对账和风控；
- 优惠券、积分、促销、评价、收藏、推荐和客服；
- 可执行 SQL 的数据库实验控制台；
- 为后续阶段提前抽象营销、物流或售后模块。

## 4. API 补齐范围

Stage 4 已提供地址、购物车、结算、订单列表、支付、取消、店主子订单、审计日志和 Top 10 API。Stage 5 只新增两个履约动作：

| 方法 | 路径 | 角色 | 行为 |
|---|---|---|---|
| POST | `/owner/shop-orders/:shopOrderNo/ship` | 店主 | 将本店 `PENDING_SHIPMENT` 子订单更新为 `SHIPPED` |
| POST | `/member/shop-orders/:shopOrderNo/confirm` | 会员 | 将自己订单下的 `SHIPPED` 子订单更新为 `COMPLETED`，全部子订单完成后总订单更新为 `COMPLETED` |

错误语义：

- 子订单不存在或不属于当前用户/店铺：`404 NOT_FOUND`；
- 状态不允许操作：`409 ORDER_STATE_CONFLICT`；
- 未登录或角色不足沿用既有 `AUTH_REQUIRED` 和 `FORBIDDEN`。

## 5. 前端页面范围

### 5.1 会员

会员侧边栏新增 `购物车与订单`。该页面包含：

- 最小地址表单和地址列表；
- 按商品卡片加购；
- 购物车列表、数量调整、删除和总金额；
- 使用默认或选中地址结算；
- 订单列表、模拟支付按钮；
- 已发货子订单的确认收货按钮。

商品目录仍是会员入口首页。商品卡片只增加 `加入购物车` 按钮，不做详情页。

### 5.2 店主

店主侧边栏新增 `订单履约`。该页面包含：

- 本店子订单列表；
- `PENDING_SHIPMENT` 子订单的发货按钮；
- 状态、金额、商品数量和所属总订单号展示。

### 5.3 管理员

管理员侧边栏新增 `数据库证据`。该页面包含：

- 审计日志最近 50 条；
- 有效销量 Top 10；
- 简短说明这些数据来自触发器、视图和窗口函数。

页面只读，不提供 SQL 输入框或修改按钮。

## 6. 测试与验收

### 6.1 API 集成测试

- 店主只能发货本店 `PENDING_SHIPMENT` 子订单；
- 非店主或其他店主不能发货；
- 会员只能确认自己已发货的子订单；
- 重复发货、未支付发货、重复确认收货返回 `ORDER_STATE_CONFLICT`；
- 全部子订单确认后，总订单状态变为 `COMPLETED`；
- 发货和确认收货产生审计日志。

### 6.2 前端组件测试

- 会员目录商品卡可调用加购 API，并展示友好提示；
- 会员购物车与订单页可创建地址、结算、支付和确认收货；
- 店主订单页只对待发货订单显示发货按钮；
- 管理员数据库证据页展示审计日志与 Top 10，不出现 SQL 输入框。

### 6.3 E2E 主线

在 Docker 演示环境中：

1. 管理员创建分类；
2. 两个店主分别发布商品；
3. 会员搜索商品并加入购物车；
4. 会员创建地址并跨店结算；
5. 会员模拟支付；
6. 两个店主分别发货；
7. 会员确认收货；
8. 管理员查看数据库证据。
