# 索引优化证据

## 目标

验证关键查询具备合适索引，并在确定性数据规模下保存 `EXPLAIN ANALYZE`、计时和结论。

## 已采集索引清单

### products

```text
PRIMARY(id)
idx_products_category_status_id(category_id, status, id)
idx_products_shop_status_updated(shop_id, status, updated_at, id)
idx_products_status_updated(status, updated_at, id)
ft_products_name_description FULLTEXT(name, description) WITH PARSER ngram
```

### master_orders

```text
PRIMARY(id)
uq_master_orders_order_no(order_no)
uq_master_orders_buyer_checkout(buyer_user_id, checkout_token)
idx_master_orders_buyer_created(buyer_user_id, created_at, id)
fk_master_orders_address(address_id)
```

### shop_orders

```text
PRIMARY(id)
uq_shop_orders_order_no(shop_order_no)
idx_shop_orders_master(master_order_id)
idx_shop_orders_shop_status_updated(shop_id, status, updated_at, id)
```

### audit_logs

```text
PRIMARY(id)
idx_audit_logs_created_id(created_at, id)
idx_audit_logs_actor_created(actor_user_id, created_at)
idx_audit_logs_table_record(table_name, record_id)
```

## 查询与执行计划

```sql
EXPLAIN ANALYZE
SELECT p.id, p.name, p.price
FROM products p
WHERE p.category_id = ? AND p.status = 'PUBLISHED'
ORDER BY p.id
LIMIT 20;

EXPLAIN ANALYZE
SELECT *
FROM master_orders
WHERE buyer_user_id = ?
ORDER BY created_at DESC, id DESC
LIMIT 20;

EXPLAIN ANALYZE
SELECT *
FROM shop_orders
WHERE shop_id = ? AND status = 'PENDING_SHIPMENT'
ORDER BY updated_at DESC, id DESC
LIMIT 20;

EXPLAIN ANALYZE
SELECT *
FROM audit_logs
ORDER BY created_at DESC, id DESC
LIMIT 50;
```

实际 `EXPLAIN ANALYZE` 摘要：

```text
分类商品分页
Index lookup on p using idx_products_category_status_id (category_id=1, status='PUBLISHED')
actual time=0.0425..0.0446 rows=20

会员订单分页
Index lookup on master_orders using idx_master_orders_buyer_created (buyer_user_id=156) (reverse)
actual time=0.0151..0.0283 rows=20

店主待发货订单分页
Index lookup on shop_orders using idx_shop_orders_shop_status_updated (shop_id=6, status='PENDING_SHIPMENT') (reverse)
actual time=0.202..0.206 rows=20

审计时间线
Index scan on audit_logs using idx_audit_logs_created_id (reverse)
actual time=0.0183..0.0476 rows=50
```

结论：

- 四条高频查询均命中预期索引；
- `EXPLAIN ANALYZE` 的实际时间来自当前 Docker/MySQL 本地环境，不外推为生产性能；
- 本次没有删除索引做前后对比，避免破坏当前开发库结构；报告中只可陈述“命中预期索引和当前实际时间”，不能声称具体提升百分比。
