# 审计日志证据

## 目标

验证关键 DML 自动写入审计日志，并证明敏感字段不会出现在审计 JSON 中。

## 已采集：商品改价审计

执行商品直接改价后，审计日志输出：

```text
id   actor_user_id  request_id                              table_name  record_id  action  old_price  new_price
132  4              00000000-0000-4000-8000-000000000603    products    3          UPDATE  "19.91"   "19.90"
131  4              00000000-0000-4000-8000-000000000603    products    3          UPDATE  "19.90"   "19.91"
```

结论：

- 成功 DML 有审计记录；
- actor 和 requestId 能对应连接级上下文；
- 商品审计 JSON 只包含价格、库存、状态、分类、名称和图片路径，不包含密码、AES Key 或手机号明文。

## 脱敏扫描

执行 SQL：

```sql
SELECT COUNT(*) AS audit_rows_containing_sensitive_words
FROM audit_logs
WHERE CAST(old_data AS CHAR) REGEXP 'password|PHONE_AES_KEY|13900000001'
   OR CAST(new_data AS CHAR) REGEXP 'password|PHONE_AES_KEY|13900000001';
```

实际输出：

```text
audit_rows_containing_sensitive_words
0
```

## 回滚不伪记

库存不足回滚证据见 `05-事务与并发控制.md` 和 `01-存储过程.md`。失败 checkoutToken `00000000-0000-4000-8000-000000000602` 未生成订单；因此也没有对应订单状态成功审计。

结论：

- 成功 DML 有审计记录；
- 失败结算不生成成功订单，也不会留下订单状态成功审计；
- 当前敏感词扫描未发现密码、AES Key 或手机号明文。
