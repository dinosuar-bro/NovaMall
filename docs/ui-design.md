# NovaMall 前端与交互设计

## 1. 设计目标

NovaMall 的视觉主题为“邻里星市”：以纯白和清醒中性表面承载内容，暖珊瑚用于主要操作和当前状态，辅以少量清泉青绿。品牌气质亲和、温暖、活力，但不使用奶油米色、促销式高饱和堆叠或与任务无关的复杂动画。根目录 `PRODUCT.md` 和 `DESIGN.md` 是后续前端实现的权威设计依据。

## 2. 三套布局

### 会员商城

- 响应式顶部导航：Logo、分类、搜索、购物车、用户菜单；
- 首页：主视觉、分类入口、热销 Top 10、商品列表；
- 移动端将导航收拢，保留搜索、购物车和个人中心主要入口；
- 商品卡片突出图片、名称、店铺、价格和库存状态。

### 店主后台

- 桌面端侧边栏：经营概览、店铺资料、商品、订单、退款、价格历史；
- 顶栏显示当前店铺和账号；
- 列表页优先使用筛选、状态标签、分页和明确批次；
- 高风险操作使用确认对话框，不使用仅靠颜色区分的状态。

### 系统后台

- 侧边栏：平台概览、开店审核、用户、分类、订单、审计、数据库演示；
- 审核页面并排展示申请内容、申请人和审批操作；
- 审计页面支持按时间、操作者、对象和动作筛选。

## 3. 页面清单

### 公开与会员页面

- 注册、登录；
- 首页、分类商品、搜索结果、商品详情；
- 购物车、结算确认；
- 订单列表、总订单详情、子订单详情；
- 退款申请和退款记录；
- 个人资料、地址管理；
- 开店申请与审核结果。

### 店主页面

- 经营概览；
- 店铺资料；
- 商品列表、新增和编辑；
- 子订单列表和详情；
- 退款审核；
- 商品价格历史。

### 管理员页面

- 平台概览；
- 开店申请列表和审核；
- 用户与角色查询；
- 分类管理；
- 平台订单只读查询；
- 审计日志；
- 数据库技术演示。

## 4. 关键交互

### 购物车

按店铺分组展示，但会员一次可结算全部店铺。数量修改使用乐观界面反馈，服务端失败时回滚并显示原因。

### 结算

展示收货地址、按店铺分组的商品和总金额。提交时生成并复用 `checkoutToken`；请求进行期间禁用重复按钮。网络失败后重试仍使用同一 Token。

### 订单

总订单展示支付和整体状态，子订单分别展示店铺、履约和退款状态。只有当前状态允许的操作才显示，但后端仍执行完整校验。

### 表单

- 字段旁显示可操作的校验提示；
- 服务端字段错误映射回对应输入；
- 未保存离开时提示；
- 提交期间显示明确状态并避免重复提交。

## 5. 状态设计

每个数据页面必须具备：

- 加载状态；
- 空状态；
- 错误状态与重试；
- 权限不足；
- 数据已被并发修改的冲突提示；
- 操作成功反馈。

## 6. 可访问性

- 所有交互可通过键盘完成；
- 使用语义化 HTML 和可见焦点；
- 表单标签与输入正确关联；
- 状态不只依赖颜色；
- 文字和背景满足合理对比度；
- 图片提供替代文本；
- 动画尊重 `prefers-reduced-motion`。

## 7. 响应式边界

- 会员商城覆盖手机、平板和桌面；
- 店主与管理员后台保证常见桌面宽度，窄屏可查看但不为复杂表格提供完整移动交互；
- 不为后台制作独立移动应用。

## 8. 前端数据边界

- 不在前端保存密码、AES 密钥或数据库信息；
- 不使用 localStorage 保存身份令牌；
- API BIGINT ID 保持字符串；
- 金额以字符串接收并通过金额工具格式化；
- 服务端返回 DTO，不直接绑定数据库字段对象。
