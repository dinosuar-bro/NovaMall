# 全文检索证据

## 目标

验证中文商品检索使用 MySQL ngram FULLTEXT，并与 `LIKE` 对照执行计划、计时和语义差异。

## 已知实现

```sql
FULLTEXT KEY ft_products_name_description (name, description) WITH PARSER ngram
```

查询形式：

```sql
MATCH(name, description) AGAINST (? IN NATURAL LANGUAGE MODE)
```

## 数据规模

```text
products  10022
```

其中 10,000 条为 Stage 6 确定性商品数据，关键词覆盖苹果、手机、茶叶、电脑、图书。

## 复现对照 SQL

```sql
EXPLAIN ANALYZE
SELECT p.id, p.name, p.price
FROM products p
WHERE p.status = 'PUBLISHED'
  AND MATCH(p.name, p.description) AGAINST ('苹果' IN NATURAL LANGUAGE MODE)
ORDER BY MATCH(p.name, p.description) AGAINST ('苹果' IN NATURAL LANGUAGE MODE) DESC, p.id DESC
LIMIT 20;

EXPLAIN ANALYZE
SELECT p.id, p.name, p.price
FROM products p
WHERE p.status = 'PUBLISHED'
  AND (p.name LIKE '%苹果%' OR p.description LIKE '%苹果%')
ORDER BY p.created_at DESC, p.id DESC
LIMIT 20;
```

结论：

- FULLTEXT 查询确实使用 `ft_products_name_description`；
- LIKE 对照没有使用全文索引，而是先用 `idx_products_status_updated` 过滤上架状态再做字符串过滤；
- 在当前合成数据中，LIKE 的实际时间并不比 FULLTEXT 慢，不能写成“全文索引一定更快”；
- FULLTEXT 的价值在于中文 ngram 分词和相关性排序，LIKE 是子串匹配，两者语义不同。
