# 数据库实验与证据计划

## 1. 目的

本文件规定 9 项高阶数据库技术的验证方法和证据格式。实验结果必须在实现后真实运行生成，禁止提前填写性能数字。

## 2. 实验环境记录

每次正式实验记录：

- 日期与 Git 提交；
- macOS、Docker Desktop、CPU 和内存；
- MySQL 完整版本与容器镜像摘要；
- 数据库字符集、排序规则和 `ngram_token_size`；
- 数据库迁移版本；
- 各核心表行数；
- 冷启动或预热状态。

## 3. 测试数据

### 功能数据集

用于正确性测试：

- 1 个管理员；
- 3 个会员，其中 2 个审核为店主；
- 店主共享平台商品池；
- 至少 3 个分类和 20 个商品；
- 可覆盖统一购物车、库存不足、订单履约和退款的订单数据。

### 性能数据集

使用确定性种子生成，不包含真实个人数据。建议最低规模：

- 100,000 个商品；
- 10,000 个会员；
- 100,000 个总订单；
- 200,000 个子订单；
- 500,000 个订单明细；
- 1,000,000 条审计日志。

如果本机资源不足，可按固定比例缩减，但报告必须记录真实规模。

## 4. 实验一：存储过程输入/输出与回滚

### 成功用例

准备包含多个商品的购物车，调用 `sp_checkout_cart`。

验证：

- OUT 参数返回订单号；
- 只创建 1 个总订单和 2 个子订单；
- 明细快照正确；
- 库存按数量扣减；
- 购物车清空；
- 支付记录为待支付。

### 失败用例

将其中一个商品库存设为不足后调用。

验证订单、明细、库存、支付和购物车均保持调用前状态，错误码为 `OUT_OF_STOCK`。

### 幂等用例

以同一 `checkout_token` 重复调用，验证返回同一订单号且不重复扣库存。

## 5. 实验二：触发器

直接执行商品改价和订单状态更新，不通过 API。

验证：

- 价格历史自动产生且新旧价格正确；
- 审计日志自动产生；
- 操作者和 requestId 与连接上下文一致；
- 审计 JSON 不含敏感字段。

## 6. 实验三：视图

- 比较 `v_member_order_details` 与基础表查询结果；
- 验证 `v_effective_product_sales` 排除取消和退款；
- 验证 `v_shop_sales_summary` 只聚合目标店铺有效交易。

保存视图定义、示例查询与结果截图/文本。

## 7. 实验四：索引优化

为每条高频查询保存：

1. 查询 SQL 与参数分布；
2. 数据规模；
3. 无目标索引时的 `EXPLAIN ANALYZE`；

重点查询：

- 店主待发货订单分页；
- 审计日志按时间和操作者查询。

## 8. 实验五：事务与并发

### 最后一件库存

设置商品库存为 1，使用两个独立数据库连接和不同购物车并发结算。

期望：

- 一个事务成功，一个事务返回库存不足；
- 最终库存为 0；
- 只有一个有效订单明细；
- 不出现负库存和半订单。

### 故障注入

在测试环境令结算过程在创建子订单后抛出异常，验证全部写入回滚。故障开关不得存在于生产迁移。

### 死锁测试

构造两个购物车以相反顺序包含相同商品，验证过程仍按商品 ID 升序加锁。记录是否发生死锁及处理结果。

## 9. 实验六：窗口函数

构造销量相同、取消和退款订单混合的数据。

验证：

- 只统计有效销量；
- 返回不超过 10 个商品；
- 按销量降序；
- 销量相同时按商品 ID 升序；
- 多次执行结果稳定。

## 10. 实验七：审计日志

以会员、店主、管理员分别执行授权操作，并执行一次越权失败操作。

验证成功 DML 有对应审计记录；失败或回滚事务不留下虚假成功日志；管理员查询可按时间、操作者和表过滤。

## 11. 实验八：全文检索

在 100,000 条中文商品数据上选择至少 5 组关键词，比较：

```sql
name LIKE '%关键词%' OR description LIKE '%关键词%'
```

与：

```sql
MATCH(name, description) AGAINST ('关键词' IN NATURAL LANGUAGE MODE)
```

保存执行计划、实际时间、返回数量和相关性排序。报告明确说明 LIKE 子串匹配与全文相关性搜索语义不同，不能只比较速度而忽略结果差异。

## 12. 实验九：数据加密

验证：

- 同一手机号使用不同 IV 后密文不同；
- 直接查询 `phone_cipher` 无法读取手机号；
- 正确密钥和 IV 可恢复原值；
- 错误密钥不能得到有效手机号；
- 审计日志、错误日志和 API 非授权响应不出现手机号明文；
- 订单中的密文快照可由授权履约接口解密。
