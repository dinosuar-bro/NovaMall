# 事务与并发证据

## 目标

验证结算、失败回滚和并发库存控制不会产生半订单或负库存。

## 已采集：库存不足回滚

存储过程证据见 `01-存储过程.md`。核心结果：

```text
orders_before
0

stock_before
19

ERROR 1644 (45000): OUT_OF_STOCK

orders_after
0

product_id  cart_quantity_after
3           999999

stock_after
19
```

结论：

- 失败时未创建订单；
- 库存未扣减；
- 购物车仍保留失败项；
- 没有半提交。

- 失败用户购物车仍保留，未产生半订单。
